## Фаза 3b — реализация (2026-07-10)

**Позиция-осведомлённый кит + пулинг в симуляции** (то самое «осознанное ограничение» инкремента 1 снято):
- Модель: `AssemblyEngineBrandKit.positions?: AssemblyKitPosition[]` — позиции с ≥2 вариантами (основной первым, запасные по убыванию стока на старте отчёта). `parts` остался плоским default-видом для легаси-потребителей (дефицит-рекомендации, shortage-тексты, номинальные потребности горизонта) — их семантика не менялась.
- Симуляция (`assemblyForecast.ts`): `unitizeKit` раскладывает кит на единицы потребления (одиночки + позиции); `maxEnginesForKit` считает ёмкость позиции пулингом (`Σ floor(сток_варианта/qty)`); `consumeOneEngineAndFormatSummary` на КАЖДЫЙ двигатель выбирает вариант (первый по предпочтению, чей остаток закрывает qty) → «основного мало, но не 0» закрывается пулингом, смена варианта адаптивна по мере расхода внутри горизонта; `requiredParts`/`variantKey` несут фактически выбранную деталь (наряд из прогноза корректен); в расшифровке помета «— запасной вариант (вместо «X»)». `consumeKit` (виртуальное списание) пулит по-вариантно и возвращает тоталы для зеркала в warehouse-bins (`takeFromBinsByTotals`; `allocateKitConsumptionFromBins` удалён как мёртвый).
- Backend (`collapsePositionsToDefaultOption`): возвращает `{chosen, positionGroups}`; kit-builder прикрепляет `positions` к киту. Инкремент-1 подстановка в `parts`-виде и warning сохранены как были.
- Тесты: shared +6 (пулинг в дне / адаптивная смена по горизонту / оба пусты → недобор / разные qty вариантов / легаси без positions / bins-подписи запасного), backend +3 (порядок options по стоку / одиночки без positions / без stock-мапы порядок стабилен). Все прежние — без изменений, зелёные.

## Контекст (проверено по коду)

- Модель кита (`AssemblyEngineBrandKit.parts`, [shared/assemblyForecast.ts](../../shared/src/domain/assemblyForecast.ts)) — **плоский список деталей**, понятия «позиция с вариантами» в ней нет.
- Коллапс позиции к основному варианту (Фаза 1b) — в бэкенде: `collapsePositionsToDefaultOption` ([warehouseForecastService.ts:365](../../backend-api/src/services/warehouseForecastService.ts)) выбирает ОДНУ строку на позицию (`is_default_option`) и **выбрасывает запасные** ещё до симуляции → прогноз их «не видит».
- Симуляция ([computeAssemblyForecast](../../shared/src/domain/assemblyForecast.ts)) складо-осведомлённая и очень интрикатная (bins, incoming, virtual consumption). Трогать её — высокий риск на ядровом прод-отчёте.
- Киты строятся (стр. 709) **до** загрузки стока (стр. 720), но `incomingLines` и всё нужное доступно раньше.

## Решение (техническое, за мной — memory `delegate-technical-decisions`)

**Инкремент 1 = stock-aware выбор варианта на этапе сборки кита, буквально по формулировке:**
у позиции с ≥2 взаимозаменяемыми вариантами, если **текущий сток основного варианта = 0**, а у какого-то запасного **> 0** → в прогноз идёт запасной (с наибольшим остатком) + операторская нота-warning.

**Почему так, а не иначе:**
- **Минимальный blast-radius** на ядровом отчёте: изменение целиком в `collapsePositionsToDefaultOption` (pure) + реордер загрузки стока в оркестраторе. **Симуляция `computeAssemblyForecast` не тронута** → 0 риска для складской логики.
- **Полностью юнит-тестируемо** (pure kit-builder, без БД/сети).
- **Легаси-поведение не меняется:** параметр стока опционален; без него (все текущие тесты) выбирается основной как раньше. Если основной в стоке — тоже как раньше (нет регресса «увёл на запасной, когда основного хватало»).
- Выбор по **текущему стоку** (не incoming): буквальное «нет на складе [сейчас]», предсказуемо и без агрегации горизонта.

**Осознанное ограничение (кандидат Фазы 3b, если владелец захочет после обкатки):**
- Не покрывает «основного мало, но не 0, а запасного много» (останется основной → shortage).
- Нет **пулинга** (собрать позицию частью из основного, частью из запасного) и нет **адаптивной** смены варианта по мере расхода стока внутри горизонта.
- Полноценная адаптивная модель = позиция-осведомлённый кит + пулинг в симуляции (крупная переделка самой интрикатной функции) — сознательно отложено.
