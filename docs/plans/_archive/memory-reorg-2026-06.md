## Образец из Мозга

`brain_matrica` держит знание двухъярусно: **Tier-1 тонкий индекс** (таблица «название + статус ✅дата», читается на старте) + **Tier-2 файлы деталей** (читаются точечно). Завершённое не пилится в рабочий файл — оно в индексе со статусом и в git. **GOTCHAS** — по симптомам, индекс + короткие записи, грепаются перед долгой отладкой.

## Целевая структура (всё В РЕПО — для синка между машинами, brain #010)

| Файл | Роль | Кто читает |
|---|---|---|
| `docs/SESSION_HANDOFF.md` | Только активная нитка + следующий шаг + открытые вопросы. Без дампа завершённого. | `/start` всегда |
| `docs/CODEBASE_MAP.md` | Карта монорепо (как сейчас). | `/start` всегда |
| `docs/PENDING_FOLLOWUPS.md` | **Только открытое** (🔴/⏳/🟡/🟢) + метки старения. ✅-записи выпилены. | по требованию задачи |
| **`docs/COMPLETED.md`** (новый) | Tier-1 done-индекс: 1 строка на завершённую нитку/релиз (`✅ дата · название · релиз/PR · итог · ссылка на архив-план`). Без описаний — они в git/PR. | по требованию («это сделано?») |
| **`docs/GOTCHAS.md`** (новый) | Проектные грабли по симптомам: Tier-1 индекс + короткие записи. | грепается перед отладкой |
| `docs/plans/` | Только **активные** планы. | по ссылке из handoff |
| **`docs/plans/_archive/`** (новый) | Закрытые планы. | по ссылке из COMPLETED |
| `docs/PROJECT_STATE.md` | Архитектура/правила/контур (как сейчас). | по требованию |
| `~/.claude/…/memory/` | **Только** USER-преференции/feedback (НЕ синкается по git). Проектный архив сюда **не** кладём. | recall по релевантности |

## Принцип

- **Открытое** → лёгкие рабочие файлы (PENDING open-only, handoff, активные планы). Дешёвый `/start`.
- **Сделанное** → `git log` + тела PR (granular) + тонкий `COMPLETED.md` (навигация уровня ниток). Читается по требованию.
- **Грабли** → `GOTCHAS.md` (по симптомам).
- **Преференции/feedback** → личная файловая память.
