## Цель

Заменить ручной Word-табель полноценным DB-backed модулем в ЭРП: быстрый ввод (палитра кодов + Excel-клавиатура + дефолт-смена + заполнение диапазона), live-подсчёты, печать на одном листе A4 для отдела кадров. Коды — официальная форма Т-13.

## Что показал разбор референс-Word (цех №2, июнь 2026)

- Заголовок: «Табель учёта рабочего времени сотрудников Цеха №2 за ИЮНЬ 2026».
- Колонки: № · ФИО · дни месяца с днём недели. В старом Word **воскресенья пропущены** (1 пн…6 сб, 7 вс нет, 8 пн…). **Правка владельца: воскресенья НЕ пропускать** — показываем ВСЕ дни месяца.
- Ячейки: часы (9 — типовая смена; 8/4/6/10/11/12) ИЛИ буква (`Б` больничный, `Н` неявка). Итогов нет — всё вручную.
- ~28 строк-сотрудников.

## Решения по открытым вопросам (мои — фиксирую; HR подтвердит точные коды)

1. **Коды Т-13** — засеять стандартный Госкомстатовский набор (Я/01, Н/02, РВ/03, С/04, К/06, ОТ(О)/09, ОД/10, У/11, Б/19, Т/20, ДО/16, ПР/24, В/26, НН/30, ПВ/22, Г/23 …), справочник **редактируемый**. Точные числовые коды — из официальной формы (не из памяти); HR сверит. [[seed-defaults brain #044]] — дефолты в БД, менять backfill'ом.
2. **Старый `0`/`Н`** — старые данные **не импортируем** (модуль ведётся с чистого листа), поэтому маппинг не нужен. `Н` в Т-13 = ночные; неявка = `НН`.
3. **Режим недели / праздники** — **конфиг на цех** (5/6-дневка), по умолчанию **6-дневка** (референс-цех №2 работает пн–сб). Все дни месяца в сетке всегда; режим влияет лишь на пред-пометку `В` и на расчёт нормы. Производственный календарь РФ (праздники) — **Фаза 3** (в Фазе 1 — пред-пометка только выходных по режиму недели; праздники оператор правит вручную).
4. **Ростер** — переиспользуем реестр сотрудников ЭРП (EAV `employee`, picker [[brain #035]]). Строки табеля добавляются выбором сотрудников; табельный № и должность — снапшотом в строке. (Авто-префилл по цеху — если у сотрудника есть link на цех; опционально.)
5. **Привязка к зарплате** — **ВОПРОС ВЛАДЕЛЬЦУ.** Предлагаю **Фаза 1 = только учёт + печать/экспорт для кадров** (без интеграции с расчётом ЗП). Интеграция с зарплатой — отдельная поздняя фаза, если нужна. Это держит фичу отгружаемой и не раздувает первый заход.

## Модель данных (PG + Drizzle, аддитивная миграция `0062_*`; следующий номер — 0062)

Серверные нативные таблицы (API-driven, **без** client-SQLite-sync — как warehouse/reports):

- `timesheets` {id, workshop_id FK→directory_workshops, year, month, status `draft|closed`, week_mode `5|6`, norm_hours, created/updated_at, deleted_at}.
- `timesheet_rows` {id, timesheet_id FK, employee_id FK→entities(employee), tab_number, position, sort}.
- `timesheet_cells` {id, row_id FK, day 1..31, code text, hours numeric(4,2)}. Т-13-ячейка = **код + часы**.
- `timesheet_codes` {code PK, num_code, title, counts_as_worked bool, default_hours, color, sort, is_active} — справочник, засеять Т-13, редактируемый.
- Уникальность: один табель на (workshop_id, year, month) среди не-удалённых; одна ячейка на (row_id, day).

## Этапы (каждый — отдельный PR под зелёными гейтами; релиз — отдельно)

**Фаза 1 — фундамент + ввод (MVP, отгружаемо):**
1. **Данные + backend:** schema + миграция 0062 + seed кодов Т-13 + `timesheetService` (CRUD табеля/строк/ячеек, list по цеху×месяцу) + route + permissions. Гейты: typecheck/lint/backend-test.
2. **IPC + bridges + типы.**

4. **UI сетка ввода:** `TimesheetGridPage` — строки (picker сотрудников), колонки = все дни месяца (выходные приглушены, кликабельны), ячейка код+часы; **палитра чипов** (клик-вставка + «кисть»); **Excel-клавиатура** (цифры→часы, буквы→код с автодополнением, Enter/Tab/стрелки, Esc, Ctrl+D); **дефолт-смена** (норма на рабочие дни) + «заполнить норму всем/по строке»; **диапазон** (fill-down/right); **live-подсчёты** (по строке: Σ часов, дни по кодам, ночные/сверхуроч.; по колонке; итоги + сверка с нормой; валидация код/часы 0..24).
5. **CDP-verify** (verifier-electron): создать табель тест-цеха, добавить сотрудников, ввод чипом+клавиатурой, проверить пересчёт. Скриншоты.
