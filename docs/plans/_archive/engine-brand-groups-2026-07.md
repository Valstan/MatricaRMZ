# Группы марок двигателей (engine brand groups)

**Статус:** в работе (2026-07-07). Ветка `feat/engine-brand-groups`.

## Задача (от владельца)

Оператору нужно **группировать марки двигателей** по произвольным характеристикам (напр. «все 59-е», «84-е + 46-е»), чтобы **разом привязывать деталь к нескольким маркам** (не выбирать вручную и не путаться), указывая одно количество на двигатель. Группы создаёт сам оператор, набирает в них марки; марка может входить в несколько групп; у группы есть название и описание характеристик. В группы входят **только марки двигателей**.

## Решения владельца (2026-07-07)

- **Семантика привязки — «снимок» (разовое расширение).** Выбор группы при привязке детали = создать обычные связи деталь↔марка по всем маркам группы СЕЙЧАС, с одним кол-вом/двиг. Марка, добавленная в группу позже, деталь автоматически не получает (применить группу заново). → Никакой новой логики в расчёте «детали марки» (карточка марки, разборка, прогноз, BOM-sync не трогаются). Нет конфликтов количества.
- **Размещение — «Производство → Группы марок»**, рядом с «Марки двигателей».

## Модель данных

Новый EAV-тип сущности **`engine_brand_group`** (без DDL-миграций):
- `name` (text) — название группы.
- `description` (text) — описание характеристик.
- `member_brand_ids` (json) — массив id сущностей `engine_brand`.

M:N бесплатно: id марки может встречаться в `member_brand_ids` нескольких групп. В группу кладём только сущности типа `engine_brand` (валидация на выборе).
