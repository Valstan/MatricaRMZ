# UI-конструктор → эскизник модулей (mockup-режим)

**Status:** Ф1–Ф3 (ядро) реализованы одним PR — холст, палитра v2, связи-нити, сноски, экспорт «Описание для разработчика». Открыто: Ф4 (шаблоны, undo, сетка/прилипание, копирование блока) — по мере надобности.
**Created:** 2026-07-16
**Context:** пилот «Мои экраны» (PR #215, `shared/src/domain/uiSpec.ts`, `ScreenEditorPage.tsx`) сделал *рабочие* экраны. Владелец уточнил истинное назначение: операторы **рисуют эскизы** желаемых модулей (не рабочие интерфейсы), с подписями «что этот элемент должен делать/содержать» и связями-нитями между элементами. Эскиз — техзадание: владелец и Claude читают его и строят настоящий модуль быстрее.

## Прецеденты (что берём)

- **Balsamiq Wireframes** — палитра узнаваемых элементов-заглушек (кнопка, таблица, поле, вкладки), нарочито «эскизный» вид, чтобы никто не принял за готовый UI. Берём: палитру и принцип «это набросок».
- **Excalidraw / FigJam** — свободный холст: тащи куда хочешь, растягивай, стрелки мышкой от элемента к элементу, заметки. Берём: холст + resize + стрелки.
- **Node-редакторы (n8n, Node-RED)** — типизированные связи (порт → порт, вид связи = цвет/стиль). Берём: классификатор связей.

Готовую библиотеку целиком (Excalidraw embed) НЕ берём: тяжёлая, generic, не даст типизированных блоков со свойством «описание назначения» и нашу тему. Свой холст на уже имеющемся стеке (@dnd-kit + SVG-слой для нитей) — компактнее и полностью наш.

## Концепция

**Экран = холст (canvas), а не вертикальный список.** Каждый блок:
- свободная позиция (x,y) и размер (w,h) — drag + resize за угол;
- **подпись** (текст на элементе);
- **описание** («что должна делать / что содержит») — свободный текст, видим по клику и в режиме «показать аннотации» (нумерованные сноски как в чертеже);
- вид — эскизный (пунктир/серость), чтобы читалось как макет.

**Палитра блоков (v2):**

| Блок | Зачем |
|---|---|
| Заголовок, Текст | как сейчас |
| Кнопка | заглушка, надпись + описание действия |
| Поле ввода / Выпадающий список / Галочка / Дата | элементы форм |
| Таблица | колонки задаются списком названий через запятую |
| Список | заглушка списка, описание «что за строки» |
| Панель/Раздел | контейнер-рамка с заголовком, визуально группирует |
| Вкладки | полоса вкладок с названиями |
| Картинка/Схема-заглушка | перечёркнутый прямоугольник |
| Стикер-заметка | свободный комментарий оператора на холсте |

**Связи-нити:** мышкой от блока к блоку (режим «Связать» либо drag от краевой точки). Каждая нить имеет **тип** (цвет/стиль линии) + опциональную подпись:

| Тип | Смысл | Вид |
|---|---|---|
| Переход | клик по A открывает B | сплошная стрелка |
| Данные | B наполняется из A / зависит от A | пунктирная стрелка |
| Фильтр | A фильтрует содержимое B | штрих-пунктир |
| Прочее | свой текст | серая линия, подпись обязательна |

**Экспорт для разработчика:** кнопка «Описание для разработчика» — генерирует текстовую спецификацию (список блоков с координатами, подписями, описаниями и связями) — её оператор/владелец показывает Claude; сам spec_json тоже читается напрямую.

## Данные

`UiSpecV2` в том же `uiSpec.ts` (EAV `ui_screen.spec_json`, синк без изменений):

```ts
type UiSpecV2 = {
  version: 2;
  canvas: { w: number; h: number };            // логический размер, default 1280x800
  blocks: MockBlock[];                          // ≤ 120
  links: MockLink[];                            // ≤ 200
};
type MockBlock = {
  id: string;
  kind: 'heading'|'text'|'button'|'input'|'select'|'checkbox'|'date'|'table'|'list'|'panel'|'tabs'|'image'|'note';
  x: number; y: number; w: number; h: number;
  label?: string;                               // подпись на элементе
  note?: string;                                // «что делает / что содержит», ≤ 2000
  items?: string[];                             // колонки таблицы / названия вкладок / пункты
  parentId?: string;                            // вложенность в панель (визуальная группировка)
};
type MockLink = {
  id: string; fromId: string; toId: string;
  kind: 'navigate'|'data'|'filter'|'other';
  label?: string;
};
```

- `sanitizeUiSpec` принимает v1 и v2; v1 конвертируется в v2 при открытии в редакторе (блоки раскладываются колонкой, intent кнопки → note «переход на вкладку …»). Обратной записи v1 нет.
- Рабочие intents/виджеты v1 **упраздняются**: просмотр экрана (`UserScreenViewPage`) становится просмотром эскиза (read-only холст с аннотациями). Прав/разделов доступа не трогаем.

## Фазы

- **Ф1 — холст:** канвас-редактор (drag, resize, зум/скролл), палитра v2-блоков, свойства (подпись/описание/items), эскизный рендер, конверсия v1→v2, read-only просмотр. Самая большая фаза.
- **Ф2 — связи:** режим «Связать», SVG-нити с типами и подписями, редактирование/удаление нити, легенда типов.
- **Ф3 — аннотации и экспорт:** режим сносок (нумерация блоков + список описаний сбоку), кнопка «Описание для разработчика» (текстовый экспорт в буфер/файл), печать эскиза.
- **Ф4 (опц.) — удобства:** копирование блока, сетка/прилипание, шаблоны-заготовки («форма», «список+карточка»), undo.
