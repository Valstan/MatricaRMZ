# Конструктор → рабочие экраны: дашборд из живых виджетов

**Status:** ✅ ЗАВЕРШЁН — Ф-A (#270), Ф-B (#273), Ф-C (мультивыделение/группировка/выравнивание)
**Created:** 2026-07-19 (решение владельца: дашборд + докрутка «Моих отчётов» + полировка эскизника; полноценные пишущие формы — отклонены сейчас)

## Context

Эскизник (Ф1–Ф4, #238/#239) и «Мои отчёты» (v2026.717.729) отгружены. Владелец хочет вернуть «рабочие интерфейсы, которыми сразу пользуешься» — но без риска целостности пилота #215. Решение: **рабочий экран = сборка из готовых read-only виджетов** поверх существующего холста `UiSpecV2`. Запись данных из конструктора — нет.

Опора на существующее (карта из Explore 2026-07-19):
- Холст/спека: `shared/src/domain/uiSpec.ts` (UiSpecV2, sanitize tolerant), `MockupCanvas.tsx` (475 строк, single-select), `ScreenEditorPage.tsx` (undo/redo historyRef, snap 8px), `UserScreenViewPage.tsx` (read-only рендер).
- Отчёты: `shared/src/domain/customReport.ts` (spec v1: filters/columns/sort/limit/суммы), `customReportService.ts` (main, run/print/CSV), шаблоны per-user в settingsStore (`reports.customReportTemplates`, gate `reports.view`).
- Навигация: `App.tsx` `setTab(tabId)` — программное открытие любой вкладки; дашборд-кнопка «открыть вкладку» реализуема напрямую.

## Фазы (каждая — свой PR)

### Ф-A — живой дашборд (ядро)
1. **`uiSpec.ts`:** новый kind `report` у MockBlock + поле `reportTemplateId?`; у `button` — поле `targetTab?` (id вкладки из белого списка). Версия остаётся 2 (аддитивно, sanitize пропускает незнакомые поля мягко).
2. **Просмотр (`UserScreenViewPage`):** блок `report` рендерится живыми данными — по `reportTemplateId` берётся шаблон (`reports.customReportTemplates`), гоняется `runCustomReport`, таблица с итогами в размерах блока (скролл внутри); refresh-кнопка. Кнопка с `targetTab` — реально кликабельна → `setTab`. Нет шаблона/прав → мягкая заглушка «отчёт недоступен».
3. **Редактор:** палитра «Отчёт-виджет» (выбор из личных шаблонов в панели свойств), у кнопки — выпадающий «переход на вкладку» (белый список доступных вкладок). В редакторе отчёт-блок — эскизная заглушка с именем шаблона (живые данные только в просмотре).
4. Права: наследуются — просмотр экрана по AccessSection, данные отчёта под `reports.view` + права пресета. Ничего нового.

### Ф-B — докрутка «Моих отчётов» ✅
1. **Группировки с агрегатами:** в `CustomReportSpecV1` аддитивно `groupBy?: string` + агрегаты по числовым колонкам (sum/count/avg/min/max, default sum) — в `applyCustomReportTransform` (pure, shared) + подытоги в рендере/печати/CSV.
2. **Шаринг шаблонов:** флаг «общий» у шаблона; общие лежат в shared-scope бакете того же settingsStore-блоба (`REPORT_USER_SCOPE_FALLBACK` — прецедент уже есть); писать/удалять общие — автор или админ, читать — все с `reports.view`. Синк — штатный (settingsStore уже синкается).

### Ф-C — полировка эскизника ✅
1. Множественное выделение: marquee-рамка + shift-click; selection → `Set<id>`.
2. Групповые операции: перемещение, дублирование (Ctrl+D), удаление, стрелки-nudge — над всей выборкой; undo — один снапшот на жест.
3. Авто-раскладка: кнопка «выровнять» (прилипание к колонкам/строкам по 8px-сетке, выравнивание левых краёв выбранного).

## Не делаем (осознанно)
- Пишущие формы из конструктора (low-code запись) — риск целостности; только точечно по конкретному сценарию поля, отдельным решением.
- Несколько источников данных в одном отчёте (join) — ждёт реальной потребности.
