# UI Visual Standards (Electron Client)

Краткий стандарт визуала для карточек/секций/таблиц в клиенте `electron-app`.

## Цель
- Единый внешний вид интерфейса.
- Единый ритм отступов и типографики.
- Адаптивность (2 колонки при ширине, 1 колонка на узких экранах).
- Сохранение специфики бизнес-экранов без визуального хаоса.

## Обязательные базовые паттерны
- Для карточек сущностей использовать `EntityCardShell` с `layout="two-column"`.
- Для секций использовать `SectionCard`.
- Для обычных таблиц данных использовать `DataTable` + `ui-table`.
- Для блоков во всю ширину внутри карточек использовать `entity-card-span-full`.

## Базовые классы
- `entity-card-grid` — адаптивная сетка карточки.
- `entity-card-span-full` — растяжение секции на всю ширину.
- `ui-section-header` — единая строка заголовка секции.
- `ui-section-title` — единый стиль заголовка секции.
- `ui-muted` — подписи и вторичный текст.
- `ui-table` — базовый стиль таблиц.
- `num` — числовая ячейка/колонка (выравнивание вправо).

## Размерная шкала (S/M/L)
Определена CSS переменными в `global.css`:
- `--ui-space-1..5` — отступы.
- `--ui-title-size` — заголовок карточки.
- `--ui-section-size` — заголовок секции.
- `--ui-body-size`, `--ui-muted-size` — основной/вторичный текст.

## Правила верстки
- Не использовать фиксированную двухколоночную сетку без адаптива (`1fr 1fr`) в формах.
- Для инпут-рядов использовать `minmax(...)` и `minmax(0, 1fr)` в `gridTemplateColumns`.
- Для числовых полей:
  - выравнивание вправо,
  - ограничение ширины через `min(100%, Npx)`.
- Для сложных блоков (вложения, длинные таблицы, сводки) — `entity-card-span-full`.

## Что делать при создании нового экрана
1. Обернуть экран в `EntityCardShell` (для карточки) или использовать `SectionCard` (для локального блока).
2. Все подблоки оформить как `SectionCard`.
3. Все обычные таблицы через `DataTable` (`ui-table`).
4. Проверить на узкой ширине окна (без наползания).
5. Проверить выравнивание чисел (`num`) и подписание колонок.
