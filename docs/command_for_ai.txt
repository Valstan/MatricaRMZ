Короткие команды управления ИИ-сессией (детали — в CLAUDE.md и .claude/commands/).

Начать сессию разработки:
  /start  (или фраза «начни сессию разработки»)
  -> онбординг: sync с origin, чтение источников правды, доклад состояния.

Закрыть сессию разработки:
  /close_session  (или фраза «закрой сессию» / «заверши сессию»)
  -> сохранить нитку в SESSION_HANDOFF, закоммитить+запушить ВСЁ через PR-flow
     (ADR-0002, прямой push в main запрещён); сессия не закрыта, пока sync-гейт
     не зелёный. GitHub — источник истины между машинами.

Создать релиз:
  /reliz  (или фраза «создай релиз» / «выпусти релиз»)
  -> выпуск по разделу Release process в CLAUDE.md: bump версии, changelog,
     релиз через PR -> merge -> тег -> GitHub Action собирает installer.
  Деплой на прод-сервер (git pull / build / миграции / restart) — отдельный
  осознанный шаг, выполняется только по явной просьбе, НЕ автоматически при
  закрытии сессии. Доступ к проду — по SSH-алиасу matricarmz.
