# Reports Guide

Актуальное описание пресетов отчетов, фильтров и правил человекочитаемости.

## Где находится реализация
- Пресеты и колонки: `shared/src/domain/reports.ts`
- Построение отчетов: `electron-app/src/main/services/reportPresetService.ts`
- Каталог отчетов (избранное/шаблоны/последние): `electron-app/src/renderer/src/ui/pages/ReportsCatalogPage.tsx`
- Страница шаблона отчета (фильтры/предпросмотр/экспорт): `electron-app/src/renderer/src/ui/pages/ReportPresetPage.tsx`
- Общие UI-утилиты отчетов: `electron-app/src/renderer/src/ui/utils/reportUtils.ts`
- IPC отчётов (включая избранное/историю): `electron-app/src/main/ipc/register/reports.ts`

## UX раздела «Отчёты»
- Вкладка `reports` показывает каталог в два уровня:
  - секция `Избранное` (только на верхнем уровне);
  - `Шаблоны отчётов` — крупные плитки **тем** (имя крупно, описание мелко, счётчик отчётов); клик проваливает в список шаблонов темы, «← Все темы» / `Escape` возвращают назад;
  - `Последние созданные отчёты` (наименование + дата создания, только на верхнем уровне).
- Поиск над плитками ищет **по всем темам** сразу и игнорирует выбранную тему — это штатный способ найти отчёт, живущий в двух темах.
- Темы объявлены в `shared/src/domain/reports.ts`: `REPORT_THEMES` (плитки) + `REPORT_PRESET_THEMES` (`Record<ReportPresetId, ...>` — пресет без темы не компилируется). Один пресет намеренно живёт в нескольких темах.
- Выбранная тема хранится в `App.tsx` (`reportsThemeId`), а не в странице: возврат из карточки пресета ремонтирует каталог, и локальное состояние уронило бы оператора на корень.
- Каждый шаблон открывается в отдельной странице (`report_preset`) с собственными фильтрами и кнопками действий.
- История последних отчетов хранится локально в `client_settings` (ключ `reports.presetHistory`), избранное — в `reports.presetFavorites`.

## Ключевые правила человекочитаемости
- Контрагент в отчетах должен отображаться именем, не `id`/UUID.
- Денежные поля должны форматироваться как рубли, проценты как `%`, булевы значения как `Да/Нет`.
- В итогах используется формулировка `Итого по отчету` (а не по отдельному типу сущностей).
- Под каждой метрикой итога должно быть понятное описание смысла (для бухгалтерии и руководства).

## Форматы вывода и экспорта
- Предпросмотр (отдельное окно печати).
- Печать.
- PDF.
- CSV.
- 1С XML (`КоммерческаяИнформация`, `ВерсияСхемы="2.10"`), кнопка: `Выгрузка 1С (XML)`.

## Темы и пресеты (32 шаблона, 8 тем)

Отчёт, задевающий две темы, лежит в обеих — это норма, а не дубликат.

### 🔧 Двигатели — ремонтный цикл: стадии, приход и отгрузка, комплектование, утиль
- `engine_stages` — стадии двигателей и прогресс (+ Контракты)
- `engines_list` — реестр двигателей: даты, марки, контракты, контрагенты, утиль, наличие на заводе, акт комплектности; выбор колонок для печати (+ Контракты)
- `engine_movements` — движение двигателей за период
- `engines_contracts_overview` — двигатели в разрезе контрактов (+ Контракты)
- `engine_readiness_to_assemble` — готовность двигателей к сборке (+ Склад)
- `engine_kitting` — комплектование двигателя (+ Склад)
- `assembly_forecast_7d` — прогноз сборки на 7 суток по остаткам и связям деталь↔марка (+ Снабжение)
- `scrap_register` — реестр утиля с причинами (+ Склад)
- `parts_compatibility` — совместимость деталей по маркам (+ Справочники)

### 📄 Контракты — заказчики, суммы, сроки и реквизиты ГОЗ
- `contracts_finance` — финансовая сводка (сумма, объем, сроки, ИГК, отдельный счет)
- `contracts_deadlines` — сроки и риск просрочки
- `contracts_requisites` — бухгалтерские реквизиты и полнота заполнения
- `counterparties_summary` — сводка по контрагентам (+ Справочники)
- `engines_contracts_overview`, `engine_stages`, `engines_list` — см. «Двигатели»

### 🚚 Снабжение — потребность в деталях, заявки, план закупок, приход
- `parts_demand` — потребность в деталях (утиль/недокомплект/доставлено/остаток) (+ Склад)
- `supply_fulfillment` — исполнение заявок в снабжение
- `supply_receipt_gap` — заявки без прихода на склад (+ Проверки)
- `norms_purchase_plan` — план закупок по нормам (+ Склад)
- `services_pricelist`, `products_catalog` — см. «Справочники»
- `assembly_forecast_7d` — см. «Двигатели»

### 🛠 Наряды — выполнение нарядов, затраты, выработка цехов
- `work_orders_report` — журнал нарядов: фильтры по каждому полю, выбор колонок, печать
- `work_order_costs` — выполнение нарядов и затраты
- `workshop_throughput` — выработка цехов (+ Склад)
- `defect_returns_summary` — возвраты брака из сборки (+ Склад)
- `work_order_payroll`, `work_order_payroll_summary` — см. «Зарплата»

### 💰 Зарплата — начисления по нарядам, своды по сотрудникам и подразделениям
- `work_order_payroll` — зарплата сотрудников по нарядам за период (+ Наряды)
- `work_order_payroll_summary` — свод начислений по сотрудникам и подразделениям (+ Наряды)
- `employees_roster` — см. «Справочники»

### 📦 Склад — движения деталей, остатки, оборотка, ремфонд
- `part_movement_journal` — журнал движений деталей
- `stock_turnover` — оборотная ведомость по складу
- `repair_fund_reconciliation` — сверка ремфонда: экземпляры vs остаток (+ Проверки)
- `warehouse_stock_path_audit` — аудит двойного учёта: зеркальная номенклатура (`spec_json` `source=part`) vs `part_card_id` (+ Проверки)
- `engine_kitting`, `engine_readiness_to_assemble`, `scrap_register`, `parts_demand`, `norms_purchase_plan`, `workshop_throughput`, `defect_returns_summary` — см. соответствующие темы

### 📚 Справочники — детали, услуги, товары, контрагенты, кадры, инструмент
- `employees_roster` — кадровый реестр с группировкой по подразделениям (+ Зарплата)
- `tools_inventory` — учет инструмента по подразделениям и статусу (в учете/списан)
- `services_pricelist` — прайс-лист услуг с ценами и привязкой к деталям (+ Снабжение)
- `products_catalog` — каталог товаров с артикулами, единицами и ценами (+ Снабжение)
- `parts_compatibility`, `counterparties_summary` — см. «Двигатели» / «Контракты»

### 🔍 Проверки — служебные сверки: где два учёта разъехались
- `movement_integrity_audit` — проверка целостности журнала движений
- `warehouse_stock_path_audit`, `repair_fund_reconciliation`, `supply_receipt_gap` — см. соответствующие темы

## Отчет по зарплате сотрудников
- `work_order_payroll` агрегирует выплаты из нарядов по сотрудникам за выбранный период.
- В строке отчета показываются сотрудник, конкретный наряд, дата наряда, КТУ и сумма начисления по этому наряду.
- Печать, PDF и предпросмотр используют отдельную печатную форму: шапка с перечнем ФИО (при нескольких сотрудниках), таблица «Данные отчета», таблица видов работ по нарядам (дата, вид работы, количество, цена, сумма), строка **Итог** (сумма колонки «Начислено» по отфильтрованным строкам) и блок подписей. Универсальные блоки «Итого по отчету», «Итоги по группам» и «Пояснение метрик» в UI для этого пресета не показываются (данные по-прежнему доступны в CSV/XML как у остальных пресетов).
- В групповых итогах (метаданные превью) отчет отдельно суммирует начисления и количество нарядов по каждому сотруднику — используется для экспорта и согласованности; на экране и в печатной форме выводится кастомный итог по начислениям.
- Общие итоги в метаданных превью показывают количество сотрудников, количество нарядов и суммарную начисленную сумму.

## Отчет по затратам нарядов
- `work_order_costs` показывает строки работ по нарядам: номер наряда, двигатель/изделие, марка, дата, вид работ, количество, сумма и бригада.
- Фильтры: период, марки двигателей, исполнители.
- Общие итоги: количество строк, количество уникальных нарядов, общая сумма затрат.

## Сводный бухгалтерский отчет по начислениям
- `work_order_payroll_summary` агрегирует начисления по сотруднику и подразделению.
- В строке отчета показываются: подразделение, сотрудник, количество нарядов, количество строк начислений, суммарный/средний КТУ, сумма начислений и средняя сумма на наряд.
- Фильтры: период, сотрудники, подразделения.
- Групповые итоги считаются по подразделениям; общие итоги включают сотрудников, наряды, строки начислений, суммарный КТУ и сумму начислений.

## Новые реестровые отчеты
- `employees_roster`: ФИО, табельный номер, должность, подразделение, даты приема/увольнения и статус занятости. Итоги: всего, работает, уволен, уволен за период + группировка по подразделениям.
- `tools_inventory`: табельный номер инструмента, серийный номер, подразделение, даты получения/списания и причина списания. Итоги: всего, в учете, списано + группировка по подразделениям.
- `services_pricelist`: каталог услуг с ценой и перечнем привязанных деталей. Итоги: количество услуг и средняя цена.
- `products_catalog`: каталог товаров с артикулом, единицей измерения и ценой. Итоги: количество товаров и средняя цена.
- `parts_compatibility`: строки совместимости по `part_engine_brand` (деталь, марка двигателя, номер сборочной единицы, количество, поставщик). Итоги: уникальные детали, марки и суммарное количество + группировка по маркам.
- `counterparties_summary`: по каждому контрагенту выводятся ИНН, количество контрактов, количество двигателей, сумма контрактов и средний прогресс. Общие итоги: контрагенты, контракты, двигатели, сумма, прогресс.

## Чеклист при изменении отчетов
- Добавлен/изменен пресет: обновить `shared/src/domain/reports.ts` и этот файл.
- Новый пресет **обязан** получить тему в `REPORT_PRESET_THEMES` (иначе не соберётся) и непустое `description` — оно показывается мелким шрифтом в списке темы; гарды в `shared/src/domain/reports.test.ts`.
- Добавлены новые фильтры: описать их семантику в этом файле.
- Изменили формат отображения колонок/итогов: проверить и backend (`reportPresetService.ts`), и UI (`ReportPresetPage.tsx`, `reportUtils.ts`).
- Добавили новые KPI/итоги: дать короткое бизнес-описание метрики.
