- **Окно verifier-клиента реально hidden** → `document.visibilityState === 'hidden'` весь прогон: visibility-gated поллы честно спят. В пробах форсить видимость override'ом (`Object.defineProperty(document,'visibilityState',{configurable:true,get:()=>'visible'})` + `visibilitychange`).
- **HMR не всегда доносит правки в lazy-page чанки** (страница уже загружена) — после правок renderer-страниц перед CDP-верифаем перезапустить electron, не полагаться на hot-swap.
- **Карточку услуги/товара открывать глобальным поиском** («Поиск» в шапке): вкладка «Услуги» — номенклатурный справочник (`NomenclatureDirectoryPage`), EAV-сущность из `admin.entities.create` там не видна; server-write доезжает в локальный поиск после `sync.run()` (~8с).
- **Страница «Черновики» грузит список на mount** (выучено 2026-07-10): если вкладка уже открывалась ранее в прогоне, повторный клик по «Черновики» НЕ перезагружает список (stale) → в драйвере после перехода жать кнопку «Обновить». Аналогично: глобальный `clickBtn('Сохранить')` может поймать чужую кнопку («Сохранить как черновик», диалог) — кнопки карточки скоупить внутрь `.v2-col-workspace`.
- **После закрытия карточки товара/услуги master-колонка НЕ возвращается к списку услуг** (выучено 2026-07-10): кнопка «Добавить услугу»/«Обновить» пропадает из DOM → в драйвере перед списковыми шагами снова кликать вкладку «Услуги». Плюс: серверные записи после первого save новой услуги (номенклатура через sync-push-ретраи) доезжают до ~10-12с — ассерты поллить, не бить одноразово.
