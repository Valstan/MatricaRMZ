# Машинные профили (`docs/machines/`)

Профиль рабочего окружения **одного компьютера**: порты dev-стенда, пути к инструментам, как поднимаются скиллы (`verifier-electron` и др.), машинные грабли. Цель — чтобы новая сессия на этом компе не тратила токены, заново нащупывая «как тут всё запускается», и не путалась в портах между разными компами.

## Модель

- **Один файл на hostname:** `docs/machines/<hostname>.md` (имя файла = `hostname` компа, напр. `PC40.md`).
- **Правит файл только сам этот комп.** У каждого компа свой файл → межмашинных merge-конфликтов нет (в отличие от одного общего файла с секциями).
- **Коммитится обычным PR-flow** (как прочие доки). GitHub — источник истины между машинами → профиль переживает реклон и доступен с любого компа на чтение (чужой профиль = просто справка, его не трогаем).

## Кто читает / пишет

- **Читает:** `/start` §0.5 — по `hostname` подхватывает `docs/machines/<hostname>.md` и использует значения вместо повторного нащупывания.
- **Пишет:** любая сессия на этом компе — **по мере того, как узнаёт** новое (порт, путь, рабочая/нерабочая команда, ABI-нюанс). Не копи «на потом»: выяснил → записал строкой.

## Правила

- **Без секретов.** Никаких паролей/ключей/токенов — только «как запускать» (порт, путь, команда). Секреты остаются в `*.env*` (gitignored).
- **Окружение, не проект.** Сюда — специфику железа/настройки компа. Проектное состояние (нитки/техдолг/планы) — в `SESSION_HANDOFF` / `PENDING_FOLLOWUPS`.
- **Кратко и проверяемо.** Строка = факт, который экономит время в след. сессии. Устаревшее — правь, не копи.

## Шаблон

```markdown
# Машина: <HOSTNAME>

**Hostname:** <HOSTNAME>
**OS:** <напр. Windows 11 Pro>
**Родительская папка репозиториев:** <напр. D:\GitHubReps\ — рядом brain_matrica>
**Shell:** <напр. PowerShell + Git Bash>

## Порты / сервисы (dev-стенд)
- PostgreSQL (dev): порт <...>, БД <...>, user <...> (пароль — в *.env.dev)
- Backend dev: http://127.0.0.1:<...>
- Vite (renderer dev): http://127.0.0.1:<...>
- CDP (electron): <порт> (MATRICA_CDP_PORT)

## Инструменты / пути
- psql.exe: <в PATH? путь?>
- <прочее: node/pnpm нюансы, утилиты>

## Скиллы (как поднимать на этом компе)
- verifier-electron: <состояние .env.dev, команды запуска, ABI-нюансы>
- <другие скиллы>

## Машинные грабли
- <симптом → причина → лечение>
```
