# 010 — Прогноз исчерпания сырья → опережающая закупка

**Статус:** 💬 обсуждается · 2026-06-24

## Проблема
Материалы (бронза, пруток/втулки, гильзы, кольца) кончаются внезапно: директор видит остаток, но не знает, на сколько дней хватит, и закупает реактивно — «уже кончилось». А ему нужно упреждение на срок закупки (подписать контракт, оплатить, привезти за 2–3 дня → за неделю-полторы). Считали «1.4 т хватит» — не хватило, надо было ~2.7 т.

## Решение
Программа прогнозирует дату исчерпания материала и дату закупки (с учётом срока поставки), заранее сигналит снабжению — как прогноз сборки, но по материалам.
- **Min-Max / точка заказа (ROP):** на материал задаются мин/макс и срок поставки; остаток ≤ мин → сигнал/черновик заявки.
- **MRP из плана:** потребность = план выпуска × норма расхода; дата исчерпания = остаток / суточный расход; дата закупки = дата исчерпания − срок поставки.

## Мировая практика
Reorder point, safety stock, min/max, lead-time-based reordering, расчёт потребности из плана выпуска (MRP). Оговорка: нужен достоверный суточный расход (из закрытых нарядов) и нормы расхода по маркам; начать с min/max по 5–10 критическим материалам.

## Заземление
- **Уже есть:** снабжение (заявки: черновик→одобрение→требование), склад/остатки, прогноз сборки (план выпуска), EAV (атрибуты без миграции).
- **Легко доделать:** атрибуты материала (срок поставки, мин, макс) + раздел «Прогноз исчерпания» в Снабжении (остаток, дата исчерпания, дата закупки, рекомендуемый объём) с красной подсветкой.
- **Месяц:** нормы расхода по маркам (MRP) + авто-черновик заявки при достижении минимума.

## → в программу (когда дозреет)
Раздел «Прогноз исчерпания материалов» в Снабжении на базе остатков + суточного расхода (из нарядов) и срока поставки; красная подсветка «закупить на этой неделе»; кнопка «создать заявку». Нормы расхода по BOM — следующим этапом.
