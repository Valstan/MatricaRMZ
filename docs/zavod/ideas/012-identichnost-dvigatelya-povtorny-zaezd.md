# 012 — Идентичность двигателя при повторном приезде (тот же номер — новый заезд)

**Статус:** 💬 обсуждается, разобрано в /zavod 2026-07-02 · источник: brain `2026-06-30-engine-same-number-re-arrival-identity`

## Проблема
Двигатель привозят на завод повторно — не по рекламации, а просто новый ремонт, иногда уже от **другого заказчика**, — а номер тот же, что был у нас раньше. Как оформить, чтобы не спутать с прошлыми движениями. Мировая практика (MRO): единица учёта — «заезд» (repair-case) со своим ID, номер — повторяемый атрибут; каждый приезд — новая запись, история по номеру видна перекрёстной ссылкой, авто-слияния нет.

## Заземление (проверено по программе 2026-07-02)
- **Хорошо:** внутренний ключ двигателя уже суррогатный (EAV-сущность), номер — атрибут. Рефактор первичного ключа НЕ нужен. Подсказка «такой номер уже есть» с кнопкой «Открыть» (v1.54.0) — половина нужной перекрёстной ссылки.
- **Конфликт:** три механизма исходят из «один номер = один двигатель навсегда»: (1) жёсткий запрет создать двигатель с существующим номером (v1.50.0, и офлайн); (2) серверная авто-склейка дублей (`engineDedupeService`); (3) ручной «Поиск дублей». Для случайных дублей это правильный выигрыш, для честного повторного заезда — тупик: второй заезд завести нельзя, программа заставляет переоткрывать старую карточку → смешение заказчиков/договоров/затрат.

## Решение (рекомендация /zavod, на утверждение владельцу)
Не строить полный repair-case (тяжело), а **осознанный обход запрета дублей** на приёмке. Запрет остаётся по умолчанию; при вводе существующего номера подсказка предлагает три пути:
1. **Рекламация** → та же карточка, вкладка «Рекламация» (идея 011, MVP в dev);
2. **Тот же двигатель, новый независимый ремонт** → **новая карточка** с тем же номером + флаг «повторный заезд» + мягкая ссылка «прежние заезды» (история видна, но заказчик/договор/затраты — свои);
3. **Другой двигатель, номер совпал** → новая карточка + флаг «коллизия номера».
Карточки с флагами 2/3 **исключаются из авто-склейки и «Поиска дублей»** (иначе склейка уничтожит разделение). Поиск по номеру показывает все заезды списком (свежий — сверху, старые — с пометкой «архивный заезд»).

## Оговорка
Повторные приезды пока редки («бывает») — делать по факту первого реального случая или вместе с рекламацией MVP (там всё равно трогается приёмка/список). Полный repair-case — только если повторные заезды станут массовыми.

## → в программу (после утверждения владельцем)
Трёхвариантный выбор при вводе существующего номера двигателя (рекламация / повторный заезд / коллизия); флаги «повторный заезд»/«коллизия номера» через EAV; исключение помеченных из авто-склейки и ручного поиска дублей; перекрёстная панель «прежние заезды с этим номером» в карточке. Запрет дублей по умолчанию сохраняется.
