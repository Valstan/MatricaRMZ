# 001 — Единая номенклатура: одна деталь = одно название + один артикул

**Статус:** ✅ решено (внедрение идёт) · 2026-06-12

## Проблема
Каждый цех/бригада называет детали по-своему, размеры не унифицированы. Хаос в учёте, дубли в справочнике, отчёты и остатки расщепляются.

## Решение
Принцип «одна деталь = одно название + один артикул» во всех цехах. Единый заводской справочник формируется совместно с главными инженерами цехов (источники: Иваныч, Илюса, Лилия; ведёт Николай).

## Что уже в программе
Артикул — полноправный различитель детали (v1.52.0); базовый список владельца загружен; раздел «Дубли номенклатуры» находит и сливает задвоения без потери данных. Земля догоняет программу: справочник чистится операторами по мере утверждения единых названий.
