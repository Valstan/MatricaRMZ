# 007 — Автокомплектование сборки: MVP «наряд → наличие → прогноз»

**Статус:** ✅ MVP определён владельцем (надиктовка 24.06) · обновлено 2026-06-24

## Проблема
Главный затор — начало сборки: из обезличенного ремфонда вручную подбирают совместимые детали, ходят по цехам и ищут «что есть». Программа не видит, сколько готовых деталей реально есть: мастера прячут «про запас» (страх безработицы при сдельщине), учёта в нарядах нет.

## Целевое (как раньше)
Компьютер по наличию готовых деталей и BOM марки виртуально «собирает» двигатель и раскатывает наряд с указанием, где что лежит. (Полный kitting — позже.)

## MVP, который владелец решил делать первым
Не гнаться сразу за полным автокомплектованием. Сначала — **дисциплина наличия:**
1. На **каждую** отремонтированную деталь пишется наряд → программа знает: «этот насос есть, отремонтирован, лежит где-то».
2. Прогноз сборки компонует двигатели из наличных деталей и предсказывает дефицит на 1–2 недели вперёд («колец/насосов не хватит → вызвать Байманова из отпуска»).
3. **Ничего не прятать:** если прячут — прогноз кричит «нет помп, делайте помпы», и мастер гонит выпуск.

**Явные отсечения владельца (НЕ в MVP):**
- Привязка детали к исходному двигателю — не нужна («нам похер, с какого сняли; надо найти деталь и собрать»). Нужна только для претензий/цены (Сырцова, экономисты) — отложить.
- Полная дефектовка каждой детали с номерами — отложить (опираемся на мастера разборки; номера всплывут после мойки).
- Автоматическая зарплата — потом (сначала наряды, потом зарплата).

## Мировая практика
Build-to-available / availability-driven planning (план от фактического наличия восстановленных узлов, а не от происхождения детали), учёт восстановленных компонентов как пополняемого фонда (pooling), динамический резерв на переделки (часть «готового» уходит назад). Оговорка: качество входа — без честной дефектовки (идея 008) прогноз «соберёт из брака»; поэтому хотя бы вердикт годности при закрытии ремнаряда.

## Заземление
- **Уже есть:** наряды (вкл. ремонтный), прогноз сборки на 7 дней (видит готовые + открытые ремнаряды, исключает утиль/ремфонд), кнопка «Выдать в работу» (v2026.622.1241), BOM, склад с локациями.
- **Легко доделать (это и есть MVP):** дисциплина «наряд на каждую отремонтированную деталь» — процессная, программа уже умеет; опционально — отметка годности при закрытии ремнаряда для честного прогноза.
- **Месяц:** полный kitting — кнопка «скомплектовать» + резерв по BOM + комплектовочная ведомость (идея 005 + 007).

## → в программу (готово к передаче в dev)
MVP — преимущественно процессный + лёгкая доводка: убедиться, что на любую отремонтированную деталь оформляется наряд и она попадает в наличие, видимое прогнозу сборки; при необходимости добавить отметку «годна/переделать» при закрытии ремнаряда, чтобы прогноз не завышал готовое. Полный автокомплект (резерв + ведомость) — следующим этапом.
