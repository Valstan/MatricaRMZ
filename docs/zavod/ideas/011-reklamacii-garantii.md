# 011 — Рекламационный / гарантийный учёт (будущее)

**Статус:** 💬 обсуждается, низкий приоритет (будущее) · 2026-06-24

## Проблема
Приходят претензии заказчиков (отказ после ремонта, задранная шейка, стружка в фильтре, нарушенная консервация). Сейчас — чтение бумаг и устный спор с представителем (Андрей), без письменного классификатора вины. Если паспорт с отметкой консервации потерян — нечем доказать. Тренда дефектов нет — проблема повторяется.

## Решение
1. Журнал входящих претензий с чек-листом дефектов → классификация вины (наша / заказчика).
2. Матрица консервации (тип: полгода/год/5 лет, дата, мастер) с дублем в листе отгрузки — защита от споров при утере паспорта.
3. Лог возвратов с разбором тренда раз в месяц → сигнал цеху (напр. стружка с мойки), правка процесса.

## Мировая практика
Warranty tracking, RMA / возвратная логистика, учёт сроков консервации/гарантии с напоминаниями. Оговорка: это будущее (претензии редки); фундамент уже есть — акт претензии заказчику, версии актов, фото.

## Заземление
- **Уже есть:** акты (приёмка/дефектовка/претензия заказчику), история изменений, контракты/ДС, фото.
- **Легко доделать (при первой претензии):** EAV-атрибуты на дефектовке (код смазки, анализ масла, класс вины), поле «дата консервации»; отчёт «реестр рекламаций».
- **Месяц:** отдельный модуль «Входящая претензия» (структурная форма + тренд возвратов).

## → в программу (Phase 3+, при первой реальной претензии)
Форма приёмки претензии с чек-листом дефектов и классификацией вины; отметка консервации в акте; отчёт «реестр рекламаций» с трендом. Сейчас — только зафиксировано как будущее.
