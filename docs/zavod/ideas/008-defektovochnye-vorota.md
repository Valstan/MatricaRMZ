# 008 — Дефектовочные ворота: вердикт годности до ремфонда

**Статус:** 💬 обсуждается · 2026-06-24

## Проблема
Бригады дефектовки на заводе исторически нет — дефектуют «на горячую» при разборке/сборке, на глаз (Рамиль-пенсионер по памяти размеров). В обезличенный ремфонд деталь попадает помеченной «готова» без явного вердикта; на сборку идут детали неясного статуса. Входной ОТК (комплектность) и выходной (обкатка) есть, а «ворот» перед ремфондом нет.

## Решение
Узкий пост (1 пост, 1 уважаемый человек уровня ОТК, можно на полставки), через который деталь получает вердикт **годна / в ремонт / утиль** ПРЕЖДЕ чем попасть в ремфонд как «готовая». Начать узко: только базовые детали; остальное пока без полного акта. По словам владельца — «самое контринтуитивное, но самое мощное против обезличенной кучи».

## Мировая практика
Incoming inspection gate / core grading в ремонте (реман): аттестованный инспектор даёт быстрый вердикт по чётким критериям перед пополнением фонда годных; poka-yoke на приёмке против явного брака. Оговорка: ворота не должны стать новым узким местом — держать темп (2–5 мин на деталь) или пакетный пересмотр раз в смену.

## Заземление
- **Уже есть:** акты дефектовки с галочками и ветками восполнения (Заказчик/Ремонт/Закупка), версии, фото деталей.
- **Легко доделать:** статус-вердикт на детали (годна/ремонт/утиль + инспектор + дата, через EAV) и роль «Дефектовщик/ОТК».
- **Месяц:** «акт дефектовки партии» как документ + интеграция: в «готовых к сборке» прогноза видны только детали с вердиктом «годна»; при «в ремонт» — предложить ремнаряд.

## Рекомендация (по делегированию владельца, 2026-06-24)
Не делать отдельный пост на грязной разборке и не привязывать деталь к двигателю (совпадает с MVP идеи 007). Вердикт годности ставить там, где деталь уже чистая и оценимая:
- **мелочь/узлы** (насосы, фильтры, помпы) — вердикт «годна» автоматически при **закрытии ремонтного наряда** (мастер отремонтировал → подтвердил); отдельная дефектовка не нужна;
- **базовые детали** (картер, блок, головка) — вердикт после мойки/опрессовки (их и так опрессовывают), отметка в акте дефектовки.

Так снимается противоречие «грязно vs перемешано»: нужен не отдельный инспектор на всё, а **отметка статуса в момент готовности**; «узкое окно» ОТК — только для спорных и базовых деталей. Это и есть гарантия качества входа для прогноза (007). Связка: идея 003 (базовые детали).
