## Текущая нитка

**Референс-целостность при удалении** — фича по запросу владельца: при удалении объекта показать связанные записи и предложить действие (удалить связи / оставить / заменить / отложить в «Заметки»). План: [`plans/referential-integrity-delete-2026-07.md`](plans/referential-integrity-delete-2026-07.md).

Что сделано:
- **Ф1 — универсальный сборщик входящих ссылок** ([#323](https://github.com/Valstan/MatricaRMZ/pull/323), merged): `findAllIncomingReferences` видит ссылки во всех хранилищах (EAV одиночные+массивные, `contract_sections` JSON, `meta_json` нарядов/заявок, BOM-junction), а не только одиночные EAV-линки. + shared-сборщики контракта/заявки. Read-only, тесты.
- **Ф2/Ф3 — диалог удаления + resolution-сервис** ([#324](https://github.com/Valstan/MatricaRMZ/pull/324), **открыт, ждёт решения владельца**): `resolveIncomingReferencesAndSoftDelete` (leave/remove/replace по всем хранилищам, sync-безопасно) + `DeletionIntentDialog` (4 действия, «отложить» → напоминание в «Заметки»). Подключён в удаление **контрагента**. Тесты зелёные, но **CDP-смоук диалога не прогнан** (PG-служба стоит).
- **Гард-фикс #319** ([#325](https://github.com/Valstan/MatricaRMZ/pull/325), merged + раскатан): регресс — гард валидировал `partId` наряда только по `entities`, а детали в `directory_parts`/`erp_nomenclature` → сборочные наряды с деталью не сохранялись. Резолв каталожных типов расширен.
- **Меню по ролям + видимость висячих ссылок** ([#322](https://github.com/Valstan/MatricaRMZ/pull/322), merged): на ПК вернулось полное role-based меню (планшетный пресет — только на реальном планшете, никогда для суперадмина); висячие ссылки показываются красным «выберите заново».
