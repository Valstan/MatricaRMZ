## Следующий шаг

**Приёмка Ф2 на живых клиентах — первое, что делать в следующей сессии, если появится возможность поднять PG.**

Прогнать двухклиентский сценарий (детально — [`plans/engine-reservation-f2-2026-07.md`](plans/engine-reservation-f2-2026-07.md) §Шаг 10, 8 пунктов). Ключевые три:
1. Клиент A жмёт «Взять в работу» на TEST-001 → у клиента B после pull карточка read-only, плашка с логином+ФИО, в списке значок 🔒.
2. B жмёт «Всё равно редактировать», правит, синк → баннер «двигатель занят, N изменений не приняты», и в «Критических событиях» **не** появилось `server.authz.denied`.
3. **Главная проверка фичи:** B уходит в оффлайн **до** взятия замка A, правит карточку, возвращается — правки НЕ отбиты (pre-lock grace).

Блокер тот же, что у смоука Ф1: на PC40 служба PostgreSQL требует UAC (`docs/machines/PC40.md` §PG и UAC) — нужен владелец у экрана либо пилотный планшет.

**Если что-то пойдёт не так на проде** — откат гейта резерва без передеплоя: `MATRICA_ENGINE_RESERVATION_GATE=off` в env backend-сервисов + рестарт.

Кандидаты следующих ниток (не начинать без сигнала владельца):
- **Ф-later #2 — упрощённое операторское меню планшета** (прятать бухгалтерские разделы). Нужен продуктовый выбор владельца: какие разделы оставить оператору.
- **Ф3 — оффлайн-hardening:** UI «мои неприменённые изменения» со снимком значения. Сейчас отбитая по резерву строка при следующем pull может быть перезаписана авторитетной версией — правка оператора исчезнет молча. См. PENDING §Планшетный режим.
- **Пилот:** купить x64-планшет (Intel N100/N150/N200) — инсталлятор ставится нативно, помечаешь «Это планшет» в Настройках.

## Открытые вопросы для пользователя
- **Планшетное меню оператора (Ф-later #2):** какие разделы оставить в цеху (двигатели / наряды / дефектовка / …), остальное скрыть? Определяет объём фильтрации.
- **TTL резерва 12 часов** — подтвердить по факту обкатки. 12 ч = смена, забытый замок рассасывается к утру; 24 ч переживают ночь, но блокируют соседа в начало следующей смены. Меняется одной константой `ENGINE_RESERVATION_TTL_MS`.
- Общее облачное окружение рутин Матрицы и Сарафана — оставить или разделить (рекомендация: разделить). Не срочно, тянется с прошлой сессии.
