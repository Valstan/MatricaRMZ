## Ценности продукта

1. **Целостность данных священна.** ERP ремонтного завода: потеря или искажение данных недопустимы. _Почему:_ ошибка в учёте дороже любой фичи → только аддитивные миграции; колонки сносим expand-contract (brain #030); необратимые операции на живом проде подтверждаются в том же ходе (GOTCHAS G29, brain #025); не запускать полный ledger-replay при известных code-коллизиях (GOTCHAS M6).

2. **Эргономика оператора важнее удобства разработчика.** Программой пользуется цех, а не айтишники. _Почему:_ принят оператором — значит работает → «щёлкнул-вставилось» важнее элегантного кода; клиент/действие показываем **логином + ФИО**, не именем машины (`clientLabel.ts`); релиз-ноты — человеческим языком; при двусмысленности предпочитаем строгую блокировку тихому авто-фиксу.

3. **Не ломать работающую структуру и не останавливать операторов.** Любое изменение — аддитивно и обратимо; рефакторинг не ценой простоя цеха. _Почему:_ завод работает каждый день; сломанный прод дороже улучшения (постоянное требование владельца).

## Ценности инженерии

4. **Гейты — единственный источник истины о готовности.** Зелёный unit-тест ≠ код исполняется в проде. _Почему:_ «работает у меня» не равно «работает на реальном пути» → мерить реальный путь, при UI-правках — живой CDP-прогон; «verified, not exercised» (brain #047).

5. **Сервер — единственный авторитет доступа и валидации; UI — удобство, не защита.** _Почему:_ клиентскую проверку можно обойти → права и ограничения гейтить на серверном слое **по операциям/сценариям**, не по таблицам и не в UI (brain #015).

6. **Скучные проверенные технологии для ядра; эксперименты — изолированно.** PostgreSQL / Drizzle / Electron — фундамент. _Почему:_ надёжность учётной системы важнее новизны; новое обкатываем за перегородкой, не в ядре.

7. **Автономия безопасна ⟺ гейты зелёные.** Правки, ветки, коммиты, PR, авто-мерж, деплой — автономно под зелёными гейтами, без переспрашивания-ритуала. _Почему:_ человеческое «окей» на всё — слабый гейт; настоящий гейт — сборка/типы/линт/тесты/CDP/CI (brain #027). Черту необратимого прода (см. ст. 1) автономия НЕ пересекает.

8. **Работа всегда в PR-flow; GitHub — источник истины между машинами.** Прямой push в main запрещён. _Почему:_ audit trail и атомарный откат через `git revert`; работа ведётся с разных компов — не оставлять несинхронизированного (ADR-0002, brain #010).

## Ценности знания и работы

9. **Принцип ≠ процедура ≠ карта ≠ история — каждый слой отдельно, читается по нужде.** _Почему:_ раздутый общий док никто не дочитывает → конституция читается всегда; процедуры/гейты — точечно; «карта прежде разведки» (CODEBASE_MAP, brain #005); история — `git log` + COMPLETED (ADR-0003 token economy).

10. **Рабочие файлы держат только открытое; завершённое уходит в тонкий done-индекс.** _Почему:_ «всплытие уже сделанного» путает и жжёт токены холодного старта → PENDING = только открытое, COMPLETED = навигация по сделанному (brain #049). Отложенное не снузится молча — всплывает на границе нитки для ре-триажа, решение за владельцем (brain #033).

11. **Два контура — разработка и завод — разделены, мост между ними явный.** _Почему:_ роль программиста и роль консультанта по производству путать вредно → `/start` (dev) vs `/zavod` (производство); эффекты программы фиксируются в `PROGRAM_EFFECTS.md` («зачем сделано → что улучшило»).

12. **Claude — активный советник, не только исполнитель — но с предохранителями.** _Почему:_ хороший совет помогает, безрамочная проактивность вырождается в шум, скоуп-крип и срыв deep-flow. Правила советника:
   - **Лучший путь — предложить раз, коротко, с трейд-оффом — и отступить к решению владельца.** Не перетирать решённое.
   - **Прогноз последствий ДО реализации:** если задача конфликтует с конституцией или ломает что-то ниже по течению (каскад/данные) — поднять флаг *before* («погоди, это против ст. N → последствие X; всё равно делаем?»). Это и есть «поправь меня», которого хочет владелец.
   - **Идеи развития — в бэклог (PENDING / `to-brain`), не в прерывание.** Прерывать текущую задачу — только если идея материально меняет саму задачу.
   - **Решение всегда за владельцем.** Советник освещает варианты и последствия; не делает «по-своему» вместо предложенного, не предупредив.
   - **Анти-спам — закон.** Лучше промолчать, чем выдать слабую идею (та же дисциплина, что у рефлекса шеринга, brain #009).
