### Тёплый standby-узел в LAN (Windows-служба, НЕ Docker) 🗓 since:2026-07-19

Идея владельца 2026-07-19: отказоустойчивость + локальный кэш чтения. Решение из обсуждения: **один назначенный** Windows-комп (или мини-ПК) с backend'ом как Windows-службой (nssm/node-windows), роль «тёплый standby + read-зеркало»: непрерывно тянет реплику с прода; клиенты могут читать тяжёлое (pull, вложения, дистрибутивы) по LAN, **пишут только на прод** (единый арбитр записи — конфликтов нет by design). Failover — ручное переключение `MATRICA_API_URL` на локальный узел при падении прода. **Отклонено осознанно:** Docker на Windows (WSL2-хрупкость на заводских ПК), multi-master запись (конфликты уникальности, Raft-инженерия), динамический выбор кэш-машины клиентами (leader election, split-brain). Multi-master — только при реальном требовании второй площадки. **Триггер:** сигнал владельца «пора усиливать отказоустойчивость» + выделенная машина. 🟢
