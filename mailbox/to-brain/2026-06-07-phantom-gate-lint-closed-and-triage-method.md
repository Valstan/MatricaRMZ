# Фантомный гейт lint закрыт + переносимый метод чистки warning'ов

> Follow-up к `2026-06-07-phantom-gate-lint-not-enforced.md` (та заметка вскрыла проблему; эта — закрывает и обобщает метод).

## TL;DR

«Гейт назван, но не enforce-ится» (#027 называл lint автономным гейтом, а CI-джоба не было → красный lint невидим в PR) закрыт за 3 PR. Попутно выработался переносимый **метод чистки lint-warning'ов**, который стоит применять в любом репо, а не слепо удалять/глушить.

## Как закрыли у себя (3 PR, паттерн enforcement)

1. **#231** — починили errors + **добавили недостающий CI `Lint`-джоб** (`pnpm -r lint`). Без джоба гейт фантомный.
2. **#232** — вычистили dead-код за 23 warning'ами (−116 строк).
3. **#233** — `--max-warnings 0` **в per-package `lint`-скриптах, не в CI**. Так локальный `pnpm -r lint` и CI enforce-ят идентично, планка в одном месте, CI ничего не дублирует.

Урок enforcement: гейт безопасен ⟺ (а) он реально гоняется в CI, (б) приведён к нулю, (в) ноль запинен так, что регресс валит сборку. Без всех трёх — ритуал.

## Переносимый метод: triage перед удалением (главное)

Warning «assigned but never used» — это НЕ всегда мусор. Прежде чем удалять, по каждому символу прогнали **fan-out git-history расследование** (5 параллельных агентов): *что делает → когда/каким коммитом введён → его потребитель удалён рефактором (хвост) или никогда не был подключён (спящая фича)?*

- Из 23 warning'ов: 22 — хвосты **завершённых** рефакторов (потребитель уехал на другой путь / убран по бизнес-решению), 1 — недописанная микро-оптимизация в one-shot скрипте. Спящих фич — ноль. Но это заранее не было известно — без git-history легко удалить живое или, наоборот, «разбудить» мёртвое.
- **eslint видит не весь мёртвый код.** Экспортируемая функция без потребителей `no-unused-vars` не ловит; за `unused`-локалью тянется транзитивно-осиротевший state + бесполезный сетевой запрос на каждом рендере (нашли `loadBomOccupancy`, гонявший HTTP без единого читателя результата). Чистая работа = **пройти по цепочкам**, иначе удаление flagged-символов рождает новые warning'и.

## Гочи инструмента (tool-bug workaround)

`eslint --fix` **молча ничего не делает** для правила `no-useless-escape`: оно отдаёт *suggestions*, а не auto-`fix` (и `--fix-type problem` не помогает — это не fix). Симптом: «--fix не применился, диффа нет». Лечение — ручная правка или `--fix-type suggestion` (в новых eslint). Любой JS/TS-репо на это напорется.

## Почему переносимо

GONBA/setka и любой TS-монорепо: (1) паттерн «назвал гейт → заэнфорси по-настоящему (CI-джоб + ноль + `--max-warnings 0` в скриптах)»; (2) метод «git-history triage перед удалением dead-кода, иди по цепочкам»; (3) eslint suggestion-not-fix гоча.
