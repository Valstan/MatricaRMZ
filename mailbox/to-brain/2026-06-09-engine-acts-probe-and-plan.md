# Акты двигателя: probe + фазовый план + вопросы

Probe-before-build выполнен (код картирован). Постройку не начинал. Якорь: `docs/plans/engine-acts-2026-06.md`.

## (1) Probe: что уже есть vs дыры

Ключевой вывод — **директива во многом уже реализована** прошлыми нитками:
- **Есть:** список деталей двигателя (`RepairChecklistPanel`, stage `engine_inventory`, источник — детали марки); печать **Акт комплектности** (`present` есть/нету, дефолт false → чистый лист ✔) + **Акт дефектовки**; дефект-количества (repairable/scrap/replace); **MVP-1 «Заявка в снабжение из негодных»** (`replace_qty>0`→черновик заявки); `repair_fund` локация; наряд→двигатель.
- **Дыры:** (G1) чекбоксы выбора-в-печать + массовые операции; (G2) 3-веточный workflow восполнения (заказчик/свой ремонт/закупка) — есть только «закупка» и **без гейта одобрения** перед требованием; (G3) prompt «недостача на сохранении»; (G4) исход наряда «отремонтирована→годна»; (G5) прогноз не видит repair-fund-готовность; (G6) per-part история статусов.

**Хранилище** (твой §2-предохранитель): статусы деталей — **плоские поля в строке** `operations.meta_json` (НЕ EAV двигателя). Новые per-part поля (ветка, «в ремонте») ложатся туда **без миграции**. История переходов (G6) — отдельный лог-носитель (кандидат `operations` тип `part_status_event` / audit_log).

## (2) Фазовый план (твой порядок, уточнён картой)

- **Ф1 — чекбоксы + массовые операции** (G1, чистый UI, без схемы) — дешевле всего, снимает главную боль (деталей много).
- **Ф2 — prompt недостачи на сохранении** (G3) → фиксация «нужно восполнить», **без авто-требования**.
- **Ф3 — 3-веточный workflow + явный гейт** (G2): поле `replenishment_branch` per-деталь; ветка «закупка» = MVP-1 **плюс гейт утверждения** (#025).
- **Ф4 — дефектовка: финал статусов + претензии** (зависит от вопроса 1 ниже).
- **Ф5 — ремфонд→наряд→прогноз** (G4/5/6) — самая тяжёлая: статус «в ремонте», исход наряда «годна», канал в `assemblyForecast`, per-part история.

2. **Кто утверждает** требование закупки / наряд: переиспользовать `director_approved` гейт снабжения + роль? (предлагаю да).
3. **Версионирование актов:** версии (для паспорта ремонта) vs перезапись? (предлагаю версии).
4. **Подписи:** список подписантов сейчас или плейсхолдеры + позже? (предлагаю плейсхолдеры).
