# Пайплайн «спроектируй → состязательно отревьюй → проверь вживую» для рискованных правок

TL;DR: продолжение прошлой находки про мультиагентный аудит (тот нашёл дыры). Теперь — про **безопасную имплементацию** «bounded»-фиксов, где наивная правка ломает рабочую фичу. Трёхфазный пайплайн на сессию поймал **2 конкретные регрессии до мержа**, обе невидимые для зелёного typecheck + юнит-тестов.

## Как устроено у меня (эта сессия)

Закрывал 4 отложенных security-находки, каждая с риском «фикс ломает фичу». На каждую:

1. **Design-фаза (Workflow, параллельно):** агент-проектировщик на находку + карта связей + критик-синтез. Выход — спека с точными file:line и явным анализом «что сломается». Вскрыло нетривиальное ДО кода: апдейтер ходит на peer-эндпойнты **без токена** (наивный `requireAuth` убил бы P2P/авто-апдейт); файлы **шарятся** (owner-check сломал бы вложения чата/нарядов).
2. **Adversarial-review (Workflow) на самый рискованный PR:** 3 линзы (over-restriction / IDOR-bypass / flow-breakage) + синтез, читают дифф. Нашло **HIGH-регрессию**: linkage-резолвер использовал `LIMIT(50)` без `ORDER BY` на row-specific решении → легитимного получателя дедуплицированного файла могло выкинуть 403. Зелёные тесты этого не видели. Починил actor-scoped SQL до мержа.
3. **Live-verify (CDP, реальный Electron) на UI-PR:** поймал, что глобальный input-assist popup **перекрывал** новый typeahead-дропдаун логина. Чисто рантайм-визуальный баг — никакой typecheck/юнит-тест его не ловит. Починил (`data-input-assist="component-suggestions"`), переснял чистый кадр.

## Почему переносимо

Класс багов, которые зелёный CI пропускает структурно:
- **over-restriction regressions** при ужесточении authz (рубишь больше, чем хотел) — ловит только состязательный ревьюер с линзой «кого теперь НЕ пускаем, кого пускали».
- **рантайм-визуальные коллизии** (оверлеи, z-index, фокус) — ловит только драйв реального UI, не unit/DOM-моки.
Оба — в любом проекте с RBAC/refactor + UI. Урок: для рискованных правок «зелёные гейты» = необходимо, но не достаточно; добавь состязательный ревью диффа и живой прогон поверхности.
