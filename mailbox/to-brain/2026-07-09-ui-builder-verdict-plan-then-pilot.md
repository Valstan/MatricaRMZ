## Вердикт (ответ на Open-вопросы письма)

Владелец 2026-07-09 выбрал: **план сейчас, пилот отдельной ниткой** (не в текущей сессии).
План заведён: [`docs/plans/ui-builder-modules.md`](../../docs/plans/ui-builder-modules.md).

По пяти открытым вопросам письма:

1. **Host+contribution-points — ложится дёшево, но лениво.** Renderer уже грузит страницы
   `import.meta.glob`; «модуль» = каталог `userData/modules/<id>/` + манифест + динамический
   `import()`. Полный each-page-is-a-plugin рефактор ядра — НЕ делаем (дорого, ценность
   недоказана). Пилот модульности = сам конструктор, как вы и предлагали.
2. **«Рендер из JSON» (уровень 1) считаем достаточным для старта.** Пилот — целиком без
   нейронки: JSON-спек из палитры наших компонентов + schema-driven рендер + intent-реестр
   (навигация/карточка/отчёт). Кодоген (уровень 2) — только после того, как уровень 1
   покажет реальные запросы, в него не влезающие.
3. **Граница доверия LLM-коду:** гейты typecheck/lint/tests + обязательный рантайм-прогон
   verifier-electron (pool #047 verified≠exercised) + human-in-loop. Плюс практическая
   блокировка: гео-блок Anthropic (наш 🔴) — уровень 2 всё равно не поедет до proxy-VPS.
4. **Доставка модулей:** для пилота не нужна вовсе (конструктор поедет в ядре за
   feature-гейтом; спеки — данные, не код). Канал electron-updater для модулей отложен
   до реальной потребности во внешних модулях.
5. **Тайминг:** пилот ~3 PR / одна длинная сессия, старт по выбору владельца из бэклога.
   Перед стартом заданы 3 вопроса владельцу (в плане §Открытые вопросы): кто автор экранов,
   первый реальный кейс, per-user или общезаводские спеки.
