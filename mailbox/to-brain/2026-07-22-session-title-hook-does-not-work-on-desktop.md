# Рецепт #081 не работает в десктопном приложении — отозвать у остальных проектов

## Что произошло

Идея #081 («имя сессии через SessionStart-хук, `hookSpecificOutput.sessionTitle`, т.к.
`set_session_title` недоступен для текущей сессии») была внедрена в MatricaRMZ 2026-07-21
(PR #296). В следующей же сессии выяснилось: **имя не меняется, сессии остаются «New session»**.

## Диагностика (проведена 2026-07-22)

Хук временно дополнен записью в лог с дампом stdin. Результат после двух холодных стартов:

```
08:38:03 title=Матрица РМЗ 22 июля stdin={"session_id":"8a99c892-…","hook_event_name":"SessionStart","source":"startup"}
08:38:15 title=Матрица РМЗ 22 июля stdin={"session_id":"62101efe-…","hook_event_name":"SessionStart","source":"startup"}
```

- Хук **вызывается** на каждом старте (`source: startup`), exit 0, JSON корректный —
  версия «не сработал matcher» отпадает.
- Тем не менее в реестре сессий приложения обе записи остались `"title": "New session"`.

**Причина видна по id.** Приложение ведёт сессии под собственными ключами вида
`local_<uuid>`, а хуку на stdin приходит другой `session_id` (голый uuid). Десктопная
оболочка заводит свою сессию поверх CLI-сессии и вывод хука в свой заголовок не переносит.
Поле `sessionTitle` работает в CLI, но на десктопе игнорируется.

## Что рабочее есть (и почему не заменяет хук)

MCP-инструмент `set_session_title` переименовывает сессию успешно (проверено). Но по
контракту он принимает **только чужую** сессию — «Must not be the current session».
Значит сессия не может назваться сама при старте; переименовать её способна лишь соседняя
сессия постфактум. Как автозамена хуку не годится.

## Решение владельца

Отозвать фишку: неработающий хук на каждом старте — лишняя нагрузка команды без отдачи.
В MatricaRMZ удалено (эта ветка): запись `SessionStart` из `.claude/settings.json` +
скрипт `.claude/scripts/session-title.js`. Оставшийся SessionStart-хук — только `git_sync_check -Warn`.

## Просьба к brain

1. Пометить #081 как **не переносимый на десктоп** (или отозвать целиком, если проекты
   работают преимущественно из приложения) и снять из рекомендаций для GONBA/setka.
2. Если где-то уже внедрено — вычистить, как здесь.

Условие применимости, если #081 всё же оставлять для CLI-контуров: рецепт годен **только**
при запуске из терминального `claude`; в десктопном приложении проверять не по «хук отработал»,
а по фактическому заголовку в списке сессий.
