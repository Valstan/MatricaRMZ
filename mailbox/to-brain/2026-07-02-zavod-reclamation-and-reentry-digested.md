# /zavod 2026-07-02: рекламация + повторный заезд — решено владельцем, в dev

## Что решено

- **Рекламация (идея 011): MVP урезан владельцем осознанно.** Та же карточка, флаг «рекламационный» → синяя точка, **вкладка «Рекламация»** с циклом (дата приёмки → причина со слов заказчика → вердикт вины [наша/заказчика/не подтвердилось] → отремонтирован → дата отправки) + свободный комментарий. Полный каркас (warranty-наряды, бакет затрат, цепочка паспортов) принят как направление, отложен; MVP ему не противоречит. Попутно владелец решил реорганизовать карточку двигателя на вкладки (4–5, антропологическая группировка).
- **Повторный заезд (идея 012): трёхвариантная схема утверждена.** Твой каркас подтвердился с поправкой: **PK-рефактор не нужен** — ключ у нас уже суррогатный (EAV), номер — атрибут. Реальный конфликт оказался не в ключе, а в **dedupe-машинерии**: жёсткий блок дубля номера (v1.50.0) + серверная авто-склейка + ручной «Поиск дублей» насильно склеивают честный повторный заезд. Решение: блок остаётся дефолтом, при существующем номере — выбор из трёх (рекламация → та же карточка / повторный заезд → новая карточка + флаг + cross-ref / коллизия → новая карточка + флаг), помеченные исключаются из склейки. Полный repair-case — только при массовости.
- Обе нитки сцеплены в одну dev-задачу («Рекламация MVP + вкладки карточки двигателя», PENDING_FOLLOWUPS; PR #31/#32/#34).

## Находка для pool (фильтр #009: значимо+переносимо+неочевидно)

**«Anti-dupe hardening ломает легитимные повторы natural key»:** когда проект вводит защиту от дублей по natural key (unique-guard + авто-склейка фоном), любой последующий переход к «повторяемому ключу» (repair-case, повторный заезд, re-enrollment) требует **exclusion-флагов во ВСЕЙ dedupe-машинерии**, не только снятия input-блока — иначе фоновая авто-склейка молча уничтожает осознанное разделение. Кандидат в pool рядом с твоим «natural-key не уникален во времени → суррогатный case-ID + cross-ref» (это его обратная сторона: не «как хранить», а «что уже настроенный dedupe с этим сделает»).
