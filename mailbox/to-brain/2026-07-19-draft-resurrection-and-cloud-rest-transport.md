# Две находки: воскрешение soft-delete поздним upsert'ом + REST-транспорт для облачных рутин

## 1. Debounced-автосейв воскрешает soft-deleted строку после commit-clear (класс гонки)

Инцидент (данные): сохранённый наряд всплывал «несохранённым черновиком» и терял номер навсегда.
Механика: debounce-автосейв черновика (1.5с) уже улетел в IPC → пользователь жмёт «Сохранить» →
commit + `clearDraft` (soft-delete) → **запоздавший upsert автосейва ставит `deletedAt: null` обратно**.
Плюс второй слой: update-путь сервиса слепо доверял полю из payload (номер), так что реанимированный
stale-снапшот затирал присвоенный номер без пути восстановления.

Переносимый рецепт (любой app с debounced-автосейвом + commit-clear):
- держать промис in-flight записи (`draftWriteRef`) и **await его перед clear**;
- upsert-хелперы, снимающие `deletedAt`, — подозревать всегда: «save» после «clear» = воскрешение;
- иммутабельные поля (присвоенные номера/даты) сервер/сервис должен брать из **строки БД**, не из payload,
  + heal-ветка для уже сломанных строк (у нас: «№ новый» лечится следующим сохранением, вместо
  delete+recreate руками).

## 2. Облачные scheduled-агенты claude.ai: SSH не работает — транспорт только HTTPS+токен

Первый реальный запуск рутины: ephemeral-контейнер без ssh-клиента/ключей/конфига, network-политика —
allowlist **доменов** (HTTP/HTTPS), SSH-порт не проходит в принципе. Рецепт: для рутин, которым нужен
прод, поднимать **узкую REST-поверхность** (у нас `/ai-chat/routine/*`): bearer-токен из env окружения
(timingSafeEqual, 503 без env), вложения base64, read-only SQL через двойной гейт (синтаксис
SELECT/WITH-only + отдельная PG-роль SELECT-only). CLI-путь оставить обёрткой над тем же сервисом —
ручные запуски по SSH продолжают работать. Сарафану может пригодиться при расширении его рутины.

Попутное наблюдение для реестра: env-vars облачного окружения видны всем рутинам окружения —
секреты разных проектов в одном окружении смешиваются (у владельца рутины Матрицы и Сарафана
сейчас в одном), рекомендация — окружение на проект.
