## TL;DR

Рефакторинги фонового поллинга («стало тише») почти невозможно верифицировать глазами — ничего не видно. Мы доказали консолидацию поллов **переписью живых таймеров**: обернуть `setInterval`/`clearInterval` ДО загрузки приложения, атрибутировать каждый живой таймер стеком создания, и ассертить **по владельцам**, а не по количеству. Бонус: перепись нашла 3 «чужих» таймера, о которых никто не знал (страницы, HMR, heartbeat).

## Как устроено у нас

1. CDP `Page.addScriptToEvaluateOnNewDocument` инжектит обёртку до кода приложения:
   `setInterval` записывает `{delay, stack: new Error().stack}` в Map, `clearInterval` удаляет.
2. После boot+login — census: сгруппировать живые таймеры по delay, для каждого ≥15с — head стека.
3. Ассерты **по ownership**: «ровно один 15000мс (общий pulse)», «ни одного raw-таймера вне allowlist
   (vite-HMR / activityTracker / App×2 осознанных)». Слепой счёт по количеству даёт ложные фейлы.
4. Гейтинг «спит при скрытом окне» проверяется живой пробой: `Object.defineProperty(document,
   'visibilityState', …)` + `visibilitychange` → visible тикает / hidden 35с = 0 тиков / возврат
   — немедленный тик. Причём подписка на РЕАЛЬНЫЙ модульный граф приложения — в dev Vite отдаёт
   те же модули по URL (`import('/src/ui/utils/pollWhenVisible.ts')` из main-world = тот же синглтон).

## Почему переносимо

Любой SPA/Electron-проект с фоновым поллингом: техника не зависит от фреймворка (перехват на
уровне window), работает через CDP/Playwright, и превращает «ничего не сломал в невидимом
поведении» из веры в ассерт. Ownership-подход (стек создания) — ключ: без него census шумит
(HMR-пинги, сторонние либы).
