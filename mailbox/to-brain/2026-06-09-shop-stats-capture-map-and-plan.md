## (2) Дизайн точки захвата — главная находка

**Модель УЖЕ умеет workshop-attributed выпуск, и оба конца нити уже связаны в наряде:**
- `workOrderClosingService.ts:235` engine-link + `:259` жёсткий гейт `workshopId` + `:387,392` `production_release`-документ с `workshopId`+`engineId`. **Но** срабатывает только на non-Regular нарядах (на проде 0); Regular (единственные) гейт пропускают и документа не создают.
- Реальный выпуск (`enginePhaseService.ts:189`, `recordEngineStatusAudit`) ловит **актора (username), но не цех**. `actor→employee→department_id` есть; **department→workshop отсутствует** — вывести цех из «кто менял» нельзя без 1 маппинга.

**Кандидаты (дёшево→дорого):** **A** вывести цех из наряда при выпуске (связь готова) · **B** добавить department→workshop → любой lifecycle-переход выводит цех из актора (покрывает 1460 выпусков, без ручного ввода) · **C** новое явное поле на UI смены статуса (нагрузка оператора). Рекомендую **A+B**; C — только если процесс требует явного выбора. Финал — после ответов владельца.

## (3) Фазовый план

- **Фаза 0** (сразу, zero-capture): срез труда по цеху из `operations`, **явно помечен «только цеха, пишущие наряды» (=Цех №1)** — honest-MVP, не пустой дашборд (#023).
- **Фаза 1** (нельзя откладывать — не добэкфиллится): включить A (+B по ответу). Копим цех-привязку у источника.
- **Фаза 2**: материальный выпуск/межцех per-цех — оживить `production_release` или иной носитель, **зависит от Q4** (нужен ли вообще). Сейчас пусто.
- **Фаза 3**: дашборд цеха — только по наполненным срезам, расширять по adoption.

Слотинг: Фаза 0 на `operations`/`engine_brand`, не конфликтует с `parts-chain-audit`.

## (4) Вопросы владельцу (отдаю ему, не угадываю)

1. Что считать «выпуском цеха» и где это фиксируется СЕГОДНЯ (наряд / смена статуса / приёмо-передача)?
2. Откуда брать цех-привязку без дубль-учёта: наряд (A) / подразделение исполнителя (B) / новое поле (C)?
3. Почему наряды пишет только Цех №1 — остальные 6 не выписывают в системе / пишут вне её / ещё не начали? (замкнётся ли нить сама по adoption).
4. Материальный выпуск и межцех-передачи per-цех — нужны ли в учёте, где фиксируются в процессе? (Фаза 2 строится только если это реальный процесс).
