# parts-chain-audit этап B (часть): дедуп деталей применён на проде

Карта-аудита (этап A) → один из её разрывов (**G2 — дубли номенклатуры → двойной счёт**) закрыт на проде. Владелец выбрал «слить по имени»; 16 «полностью пустых» групп (мусор импорта 2026-04-16) — снести целиком.

## Результат (прод, без version bump — серверные скрипты+данные)
- **RETIRE 37** пустых стабов, **MERGE 20** пар → 12 каноничных.
- Дублей по имени деталей: **29 групп / 70 строк → 0**. Активных part-сущностей 186→129. brand-links 718→707 (8 dup + 3 dead дропнуты, переносы net-0). Бэкап `pg_dump` до apply.
- Инструмент: расширил `mergeDuplicatePart` (перенос brand-links + дедуп) + новый `retireUnusedParts` (сносит только бесссылочные, иначе репорт+скип).

## Переносимая находка (pool-кандидат): «dry-run на реальном проде ловит то, что схема не показывает»
Расширял prod-data merge-tool инкрементально. Schema-reasoning и счётчики «по верхам» были недостаточны — **dry-run на боевых данных + первый --apply поймали 3 разных бага целостности ссылок**, которых не видно из схемы:
1. **Soft-deleted владельцы ссылок:** скан `attribute_values ... deleted_at IS NULL` считал ссылки мёртвых сущностей (ретайрнутая junction-строка оставляет атрибут живым) → ложный скип/переоценка. Фикс: `JOIN entities ... e.deleted_at IS NULL`.
2. **Неполные junction-строки:** связь с `part_id`, но без `engine_brand_id` — read-API её прячет (требует оба поля), а raw-скан видит → рассинхрон счёта. Обрабатывать от набора live-рефов, не от отфильтрованного read-API.
3. **Soft-deleted FK-цель:** связь на soft-deleted марку — `upsert` отказывает «не найдено», ломая перенос на середине. Различать «есть строка» и «цель жива»; мёртвую связь дропать, не переносить.

Урок: для дедуп/merge-миграций по EAV/junction — **дробить на retire-clean + merge-with-transfer, гонять dry-run на проде, делать инструмент идемпотентным** (повторный прогон дозавершает частичную мутацию). Идемпотентность спасла: первый --apply упал на 2 парах из 20 на (3) — после фикса повторный прогон дозавершил их без ручного разбора.
