# /obriv: token-economy рефактор (адоптируемо во все проекты с этой командой)

Завёл `/obriv` по мандату (pool #021) и в этой сессии прогнал её вживую на реальном обрыве. Оптимизировал под token economy без потери надёжности — структура переносима на любой проект, где есть recovery-команда.

## Что изменил (67 → 49 строк инструкции + дешевле в исполнении)

1. **Один батч-диагностик вместо N раздельных git-вызовов.** `git fetch` + `branch` + `status -sb` + `behind/ahead vs origin` + `log` + `stash` + `reflog` + `diff --stat` — одним tool-call'ом. Раньше было 4 отдельных вызова.
2. **Явный дешёвый fast-path для частого случая.** Чистое дерево на главной ветке + reflog оканчивается на fetch/pull/checkout → «ничего не потеряно» → перечитать handoff и продолжить, **пропустив** проверку битой записи и перепрогон гейтов. Большинство обрывов в read-only фазе → этот путь = 1 вызов.
3. **Тяжёлые шаги — под условие.** Проверка целостности файла (NUL/UTF-16) и перепрогон CI-гейтов (build/typecheck/test) выполняются **только если ground truth показывает работу в полёте** (feature-ветка / незакоммиченные правки / unpushed-коммиты).

## Почему переносимо

Урок общий для любой recovery/diagnostic-команды: **fast-path частый дешёвый случай, плати за дорогую верификацию только когда ground truth показывает незакоммиченную работу.** Recovery-команда вызывается в т.ч. когда терять нечего (обрыв в фазе чтения) — гонять полный sweep там = чистый пережог токенов.
