# Tactical practices — принято, применяю

Директива `recommend` / SHOULD принята. Baseline ($325.60 / 66% / две топ-сессии по $113) понятен — да, длинные cold-start'ы это я. Гипотезу «широкая разведка codebase без оснований» признаю.

## Что меняется со следующих сессий (и уже в этой)

1. **`/start` — узкий по умолчанию.** SESSION_HANDOFF + git log + `CODEBASE_MAP.md` (создаётся параллельно этой сессией) + конкретно запрашиваемая часть кода. Никакого «прочитай весь `docs/` для понимания общей картины». Сам `/start`-skill обновляется в этом же PR — после `SESSION_HANDOFF.md` он будет читать **только** `CODEBASE_MAP.md` (карту), а три файла-источника правды `DEVELOPMENT_LOG / PENDING_FOLLOWUPS / PROJECT_STATE` — **только если активная нитка их требует** (раньше читал безусловно).
2. **Routing моделей.** Opus 4.7 1M — для архитектуры / кросс-модульных рефакторингов / ADR-уровня решений. Sonnet или `/fast` (Opus с faster output) — для мелочи (форматирование, переименования, локальные правки). Не правило «всё на Sonnet», правило подобранного инструмента.
3. **Sub-agents `Explore`.** Когда нужно >3 Glob/Grep подряд по одной теме или прочитать 5+ файлов для понимания одной фичи — делегирую агенту, не загрязняю основной контекст. Возврат — summary.
4. **`/compact` вместо новой сессии** для смежных задач (тот же модуль, продолжаем нитку). Новая сессия — когда тема другая или нужен явный сброс.
5. **Параллельные tool calls** — продолжаю. Read × N в одном message-блоке, не последовательно.
6. **`/cost` в конце значимой сессии** — добавлю в `/close_session` skill (отдельный мелкий PR или в этот же). Будет материал для второй точки замера через 2 недели.

## Что НЕ делаю (антипаттерны из письма)

- ❌ Не превращаю в формальность: если задача реально требует широкого контекста — читаю столько сколько нужно.
- ❌ Не считаю sub-agent «всегда дешевле». Узкая задача = прямой Read.
- ❌ Не игнорирую SESSION_HANDOFF.
- ❌ Не делаю «/cost-театр» — один замер в конце, не каждые 10 минут.

## Связь с CODEBASE_MAP

`CODEBASE_MAP.md` (отдельная директива, делаю параллельно) — это структурная половина того же подхода: карта снижает потребность в широкой разведке, поэтому `/start` может быть узким. Без карты `узкий /start` рисковал бы хуже знать codebase. С картой — узкий старт работает как задумано.

## Замер — согласен

Через ~2 недели (≈ 2026-06-07) `/audit-usage 14d` в brain'е сравнит с baseline 2026-05-16…05-23. Цель — top-сессии MatricaRMZ с ~$113 → ~$60-80 при том же объёме. Если эффект слабый — пересматриваем гипотезу.
