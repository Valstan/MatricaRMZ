# Находка: characterization-снимок через живое приложение как гейт распила монолита

## TL;DR

Распил 5000-строчного модуля (`reportPresetService.ts`, 27 отчётов-пресетов,
37 builder-функций) на слоёный каталог. Чисто механический рефактор: тот же
код, другие файлы. **typecheck зелёный + юнит-тесты зелёные НЕ доказывают, что
поведение не поехало** — тесты покрывали ~части, а перенос кусков между файлами
с переклейкой импортов/`export` легко роняет одну ветку молча. Гейт, который
реально ловит регрессию: **снять полный снимок выходной поверхности через
работающее приложение ДО рефактора, снять ПОСЛЕ, сравнить бит-в-бит.**

## Как устроено у нас

Отчёты строятся в main-процессе Electron из локальной SQLite-реплики. Драйвер
через CDP-bridge (`window.matrica.reports.*`) прогнал **все 25 пресетов** на
живой dev-базе (1583 двигателя, 645 строк совместимости и т.д.), сериализовал
`{columns, rows, totals, subtitle, ...}` каждого в JSON (нормализовав только
`generatedAt`), сложил в файл. Рефактор → второй прогон → `diff` двух JSON:
**25/25 идентичны**, дайджест списка пресетов идентичен. Плюс точечно пробированы
экспортные пути (CSV, 1С-XML) и надстройка («Мои отчёты»), которые ходят через
тот же движок. Итог рефактора: тонкий re-export шим на старом пути → ни один
импортёр не тронут.

## Почему переносимо

Это **характеризационное (golden-master) тестирование**, но применённое
_оппортунистически через уже запущенное приложение_, а не через заранее
написанные тесты. Ключевой момент для любого проекта, где предстоит распил
монолита, по которому нет широких тестов:

- Не «пиши сначала характеризационные тесты» (дорого, откладывает рефактор), а
  «у тебя УЖЕ есть исполнитель — само приложение; сними его выход как снимок».
- Годится для любого модуля с **детерминированной, перечислимой выходной
  поверхностью** (набор отчётов/эндпойнтов/рендеров/CLI-команд с фикс. входом).
- Дёшево: один драйвер-скрипт, два прогона, `diff`. Ловит именно тот класс
  ошибок, который typecheck+частичные тесты пропускают (потерянная ветка,
  перепутанный порядок, сорванный дефолт при переклейке импортов).
- Дисциплина: **нормализуй нестабильные поля** (таймстемпы/uuid) перед diff,
  иначе «шум» замаскирует реальный разлёт или даст ложный красный.
